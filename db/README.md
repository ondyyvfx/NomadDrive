# База данных NomadDrive

## `dump.sql` — полный дамп БД

Скрипт пересоздаёт схему, настраивает RLS и заливает чистые демо-данные
(все фотографии вручную проверены — марка авто совпадает с изображением).

### Применение

1. Откройте проект в [Supabase Dashboard](https://supabase.com/dashboard).
2. **SQL Editor → New query**.
3. Вставьте содержимое `dump.sql` и нажмите **Run**.

Скрипт идемпотентный — можно запускать повторно (он делает `DROP ... CASCADE`
перед созданием). ⚠️ Это **удаляет** существующие данные в этих таблицах.

### Что внутри

| Объект | Назначение |
|---|---|
| 9 таблиц | profiles, cars_for_rent, cars_for_sale, parts, promo_codes, bookings, orders, order_items, reviews |
| RLS-политики | публичное чтение каталогов, запись только для админа, приватные брони/заказы |
| `handle_new_user()` | триггер: автосоздание профиля при регистрации |
| `is_admin()` | проверка роли (security definer, без рекурсии RLS) |
| `verify_booking(uuid)` | публичная проверка брони по QR (security definer, `anon`) |
| Админ-юзер | готовый аккаунт администратора (см. ниже) |
| Демо-данные | 10 авто в аренду, 10 на продажу, 14 запчастей, 3 промокода |

### Готовый администратор

Дамп **сразу создаёт админа** — после `Run` раздел `/admin` доступен без
дополнительных шагов:

| | |
|---|---|
| **Логин** | `admin@nomaddrive.kz` |
| **Пароль** | `Admin12345!` |

> ⚠️ Смените пароль после первого входа (в продакшене).

Блок создания админа идемпотентный и устойчив к разным версиям схемы
Supabase Auth (учитывает наличие `auth.identities.provider_id` и тип `id`).
Если автосоздание не сработало, можно выдать права вручную:

```sql
update public.profiles set role = 'admin'
where id = (select id from auth.users where email = 'ВАШ_EMAIL');
```

Либо создать пользователя через **Dashboard → Authentication → Add user**
(с авто-подтверждением), затем выполнить `update` выше.
